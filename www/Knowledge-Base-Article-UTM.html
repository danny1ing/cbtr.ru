Отслеживание входящего трафика с помощью UTM-меток

С помощью счетчика статистики посещений, установленного на сайте, можно узнать, что посетитель пришел по ссылке из соцсети, контекстной рекламы или другого источника. Проблема в том, что неизвестно, из какого именно поста или по какому именно объявлению он пришел. Счетчики по умолчанию просто определяют источники переходов и не дают дальнейшей детализации. Эффективно решить эту проблему позволяют UTM-метки (сокращение от “Urchin Tracking Module”). Еще их называют UTM-переменными или UTM-параметрами. Они являются частью ссылки и не влияют на отображение страниц сайта, при этом позволяют получить расширенные данные о переходах по ним.

Систему этих меток разработала компания Urchin Software, которая в 2005 году вошла в состав Google. Интересно, что именно на базе программного обеспечения Urchin была запущена система Google Analytics. Компании Urchin давно нет, а вот название “UTM” с тех пор сохранилось неизменным.

Схема работы с UTM:

  1. Определить перечень ссылок, по которым необходимо получать данные
  2. Дополнить ссылки UTM-метками
  3. Разместить полученные ссылки в необходимых местах (в посте в соцсети, в объявлении контекстной рекламы, в статье в блоге, в электронном письме и т. п.)
  4. Анализировать трафик с помощью специальных отчетов в Яндекс.Метрике или Google Analytics

Для того чтобы получать данные по UTM-меткам, каких-то дополнительных действий и настроек в системах аналитики производит не нужно. Необходимо только правильно дополнить ссылку метками и опубликовать ее. Вручную составлять метки долго и трудоемко, к тому же высока вероятность ошибки. Поэтому лучше пользоваться одним из автоматизированных сервисов:

  • utmurl.ru
  • ppc-help.ru/utm_generator
  • ga-dev-tools.appspot.com/campaign-url-builder

Все сервисы работают по похожему принципу: следует задать адрес страницы на сайте и указать значения UTM-меток в предусмотренных полях. В итоге ссылка будет сгенерирована автоматически.

Логика формирования ссылок с UTM-метками:

  1. Выбирается кампания, по которой необходимо получать дополнительные данные о трафике. Допустим, это будет контекстная реклама новогодней акции в Яндексе
  2. Указывается, где будет размещена ссылка: “источник: [ЯндексДирект]”
  3. Указывается канал, по которому пользователь приходит на сайт: “канал: [реклама-с-оплатой-за-переход]” (также известный как CPC)
  4. Добавляется название кампании — оно должно быть понятным именно вам: “кампания: [новогодняя-акция]”
  5. Указывается ключевое слово (ключ), по которому показывается объявление контекстной рекламы: “ключ: [купить-елку]”
  6. Добавляется информация, например, об использовании двух заголовков в объявлении: “контент: [длинный-заголовок]”
  7. Все эти составляющие складываются в одной ссылке: examplelink.ru/“источник: [ЯндексДирект]”, “канал: [реклама-с-оплатой-за-переход]”, “кампания: [новогодняя-акция]”, “ключ: [купить-елку]”, “контент: [длинный-заголовок]”.

Ссылка с UTM-метками

https://examplelink.ru/

Ссылка

источник / канал / кампания / ключ / контент

Параметры

Вначале идет обычная ссылка вида https://examplelink.ru/ — со слешем (/) в конце. После слеша (/) идет знак вопроса (?). Далее — перечень UTM-меток (utm_source, utm_medium и т. д.). Каждая метка имеет свое значение (yandexdirect, cpc и т. д.). Оно присваивается метке после знака (=). То есть конструкция utm_source=yandirect относится к одной метке, utm_medium=cpc — к другой, utm_campaign=novogod-akcija — к третьей и т. д.; знак & разделяет UTM-метки друг от друга; в конце URL нет закрывающих знаков.

Теперь нужно применить полученную логику для создания ссылки, которую будет понимать система аналитики. URL с UTM-метками выглядит так: https://examplelink.ru/? utm_source=yandexdirect&utm_medium=cpc&utm_campaign=novogod-akcija&utm_term=kupit-elky&utm_content=long_title

Виды UTM-меток

Платные

  • utm_source

    Определяет источник трафика

    • yandex — поисковик Яндекс
    • google — поисковик Google
    • ok — соцсеть Одноклассники
    • email — рассылки по электронной почте
  • utm_medium

    Идентифицирует рекламный канал* в рамках конкретного источника переходов

    • cpc — контекстная реклама (модель оплаты за клик - Cost per Click)
    • display — баннеры
    • qr — просканированный QR-код*
  • utm_campaign

    Содержит наименование кампании или продукта, который продвигается

    • nedvizhimost_gilaya — реклама жилой недвижимости
    • iphone8 — продвижение iPhone 8
    • new_subscribers — рассылка новым подписчикам

Вспомогательные метки (ставятся по мере необходимости)

  • utm_term

    Содержит информацию о ключевой фразе, по которой пришел посетитель. Также может использоваться для идентификации других данных (размера баннера, места его размещения и т. п.)

    • kupit_iphone8 — ключевая фраза
    • 120_80 — размер баннера
    • top_left — расположение баннера
  • utm_content

    Метка для любой дополнительной информации (дата создания объявления, заголовок статьи и т. п.)

    • 02_08_2017 — дата создания объявления
    • 10_sposobov_prodvinut_sajt — название статьи
  • utm_campaign

    Содержит наименование кампании или продукта, который продвигается

    • nedvizhimost_gilaya — реклама жилой недвижимости
    • iphone8 — продвижение iPhone 8
    • new_subscribers — рассылка новым подписчикам

*указанные значения — лишь примеры, и их можно менять и дополнять по своему усмотрению. Главное - определиться со структурой, чтобы всегда можно было понять, что обозначает каждая из меток

Для корректного учета трафика по UTM нельзя допускать ошибок в синтакисе. Кроме того, необходимо включить 3 обязательные метки — даже без одной из них статистика не собирается (кроме данных, доступных по умолчанию).

Сферы применения UTM-меток и примеры

Контекстная реклама

https://examplelink.ru/?utm_source=yandex&utm_medium=cpc&utm_ campaign=krossovki&utm_term=cena_krossovki&utm_content=title_price

Трафик идет из контекстной рекламы в Яндексе (о чем говорят значения utm_source=yandex и utm_medium=cpc). Это кампания по продвижению кроссовок (utm_campaign=krossovki). Объявление показано в ответ на запрос «цена кроссовки» (utm_term=cena_krossovki). В заголовке использована цена (utm_content=title_price)

Контекстная реклама

https://examplelink.ru/?utm_source=yandex&utm_medium=cpc&utm_ campaign=krossovki&utm_term=cena_krossovki&utm_content=title_price

Трафик идет из контекстной рекламы в Яндексе (о чем говорят значения utm_source=yandex и utm_medium=cpc). Это кампания по продвижению кроссовок (utm_campaign=krossovki). Объявление показано в ответ на запрос «цена кроссовки» (utm_term=cena_krossovki). В заголовке использована цена (utm_content=title_price)

Баннерная реклама

https://examplelink.ru/?utm_source=mailru&utm_medium=display&utm_ campaign=kedi&utm_term=300_300&utm_content=main

Переходы осуществляются с баннерной рекламы (utm_medium=display) на портале Mail.ru (utm_source=mailru). Рекламируются кеды (utm_campaign=kedi). Размер баннера — 300х300 пикселей (utm_term=300_300), он размещен на главной странице (utm_content=main)

Маркетинг в соцсетях

https://examplelink.ru/?utm_source=ok&utm_medium=smm&utm_campaign= advice&utm_term=end&utm_content=10_sovetov_po_reklame_v_ok

Пользователи переходят с соцсети Одноклассники (utm_source=ok) в результате кампании в социальной сети - SMM (utm_medium=smm) - по публикации советов (utm_campaign=advice). Трафик зафиксирован по ссылке в посте на тему “10 советов по рекламе в Одноклассниках” (utm_content=10_sovetov_po_reklame_v_ok), которая расположена в конце поста (utm_term=end)

E-mail-маркетинг

https://examplelink.ru/?utm_source=email&utm_medium=digest&utm_ campaign=promo&utm_content=top

Трафик идет из рассылок на электронную почту (utm_source=email) в формате дайджеста (utm_medium=digest). Письмо посвящено промо-предложению (utm_campaign=promo). Посетитель пришел по URL вверху письма (utm_content=top)

Оффлайн-маркетинг (при условии сканирования QR-кода или SMS-рассылки)

https://examplelink.ru/?utm_source=newspaper&utm_medium=vestnik&utm_ campaign=diz_mebel&utm_content=discount10

Трафик идет после сканирования QR-кода в газете (utm_source=newspaper) под названием «Вестник» (utm_medium=vestnik). В объявлении предлагается 10% скидка (utm_content=discount10) на дизайнерскую мебель (utm_campaign=diz_mebel)

При составлении UTM-меток надо учитывать такие нюансы:

  • Параметры меток чувствительны к регистру (cpc, Cpc и CPC — это разные значения!)
  • Лучше использовать везде латиницу, чтобы системы аналитики корректно обрабатывали данные
  • В значениях UTM нельзя использовать знаки #, ?, & и =, иначе они могут быть неверно восприняты системой веб-аналитики

Недостатки UTM-меток

Появление дублирующихся страниц

В этом случае на одну и ту же страницу будет вести как минимум два URL: с UTM-метками и без них. Для поисковых систем это две разные страницы с одинаковым содержимым, из-за чего могут возникнуть проблемы при индексации.
Решение:
а) на странице, куда ведет ссылка, в разделе <head> надо указать основную ссылку, по которой страница должна быть доступна в поиске, с помощью такой записи: <link rel="canonical" href="http://site.ru/stranica.html"/>;
б) закрыть от индексации страницы с UTM-переменными. Для этого в файле robots.txt, который находится в корневой папке сайта, необходимо задать директивы clean-param*, добавив такие строки (работает только для Яндекса):

Clean-param: utm_campaign /
Clean-param: utm_medium /
Clean-param: utm_source /
Clean-param: utm_term /
Clean-param: utm_content /

Альтернатива — добавить в robots.txt строку Disallow: /*?utm_ . Так можно добиться исключения URL с UTM-метками из индексации.

Слишком длинная ссылка

Ссылка с UTM содержит много символов, из-за чего снижается ее визуальная привлекательность и ее невозможно запомнить (особенно это критично для наружной рекламы (ссылка на статью: Способы продвижения сайта через оффлайн-каналы, Этап 2) или рекламы в соцсетях (ссылка на статью: Способы продвижения сайта через социальные сети, Этап 2)).

Решение: воспользоваться одним из сервисов сокращения ссылок (bitly.com, clck.ru и др.). С их помощью длинная ссылка будет выглядеть примерно так: http://bit.ly/2uZOvB8.

Данные о трафике, который пришел по URL с метками, доступны в:

  • Google Analytics

    Источники трафика > Весь трафик > Источник / Канал

    Отчеты можно выстроить по источникам трафика, рекламным каналам, ключевым словам и другим параметрам. По каждой метке можно увидеть количество посетителей, время, проведенное ими на сайте, показатель отказов и многие другие данные.

  • Яндекс.Метрика

    Отчеты > Источники > Метки UTM

    Для корректной идентификации меток Яндекс.Метрикой они должны идти в строгой последовательности: utm_source, utm_medium, utm_campaign, utm_content, utm_term.

UTM — это простой и бесплатный инструмент, который позволяет точно идентифицировать трафик. За счет гибкости настройки меток можно узнать, по какой ссылке и из какого источника пришел посетитель. Но при составлении меток необходимо быть очень внимательными, чтобы не допустить ошибок. В противном случае, данные по UTM собираться не будут.
